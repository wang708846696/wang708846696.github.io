gradle中的写法：

这样就引用到了com.android.tools.build:gradle:3.1.2@jar中的文件com.android.application.properties，对应执行的就是文件中的类：implementation-class=com.android.build.gradle.AppPlugin

所以源头就是AppPlugin.java

然后执行AppPlugin的apply方法,直接执行BasePlugin.apply()

文档查询地址:

  • gradle-dsl
  • 变体的概念
  • Android Plugin DSL Doc

接着看BasePlugin.apply()

所以主要流程就是configureProject–>configureExtension–>createTasks

一. configureProject()

  1. 创建SdkHandler，其中根据local.properties寻找各种目录
  2. 创建androidBuilder
  3. 创建DataBindingBuilder，此类是gradle内部使用
  4. 应用将java–>class用的插件
  5. 为assemble添加注释
  6. 加载dex缓存
  7. 构建完成时候的回调，清除dex缓存
1. 创建SdkHandler
1.1 SdkHandler.findSdkLocation()
2. 创建androidBuilder

二. configureExtension()

2.1 AppPlugin.createExtension()
2.2 NdkHandler.NdkHandler()
2.3 BuildCacheUtils.createBuildCacheIfEnabled()
2.4 AppPlugin.createVariantFactory()

VariantFactory的实现，用于生成APK的项目。

2.5 AppPlugin.createTaskManager()

创建task管理类，TaskManager用于在Android应用程序项目中创建任务

2.6 VariantManager.VariantManager()

变体管理构造

三. createTasks()

3.1 build.gradle执行过程中配置的任务

评估项目的说明

这里将任务的创建分为两部分，也就是在evaluate前后。这里的evaluate可以理解为build.gradle文件执行前后 ，也就是说：方法taskManager.createTasksBeforeEvaluate()是build.gradle文件执行时调用；方法createAndroidTasks()是在build.gradle文件执行完后调用。之所以这样分步创建，是因为很多构建任务都依赖于对插件扩展对象，也就是在build.gradle文件中的android dsl部分的具体配置信息。

3.2 createAndroidTasks()
3.2.1 VariantManager.createAndroidTasks()

变体创建的入口点，其中变体工厂是在AppPlugin.createVariantFactory()创建的，在2.4详细讲解

3.2.2 createTasksForVariantData()

在createTasksForVariantScope()中才是真正干活的各种task

值得留意的是 Add a compile task这个是完成java编译的task是将java->class

这里在普及一个小知识，项目中gradle执行的时候，会先解析setting.gradle,然后是build.gradle,如果想在解析build.gradle之前做点事，可以使用project.beforeEvaluate如果想在解析build.gradle之后做点事可以project.afterEvaluate。
